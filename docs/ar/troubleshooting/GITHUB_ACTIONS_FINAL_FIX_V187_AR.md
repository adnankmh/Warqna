# إصلاح GitHub Actions النهائي — V187

## أصل المشكلة

ملف ZIP لا يحذف ملفًا قديمًا موجودًا في مجلد المستودع. لذلك كان `backend-laravel/.env` يبقى بعد فك الحزمة فوق النسخة القديمة، حتى عندما لا تحتوي الحزمة الجديدة عليه. كذلك بقي `composer.lock` قديمًا لا يحتوي PHPUnit وMockery.

## ما تغيّر

1. النسخة الكاملة الجديدة نظيفة ولا تحتوي `.env` أو مفاتيح توقيع.
2. أداة الاستبدال تحذف محتويات المشروع القديم مع إبقاء `.git`، ولذلك تظهر الملفات القديمة كحذف حقيقي في GitHub Desktop.
3. Backend CI يفحص `composer.lock`. إذا كان قديمًا أو غير متوافق، يحذفه في Runner ويعيد توليده من `composer.json` قبل الاختبارات.
4. بعد التثبيت يتم التحقق صراحة من `phpunit/phpunit` و`mockery/mockery`.
5. تم تحديث مراجع GitHub Actions الرسمية الحالية.

## تنبيه أمني

أي ملف `.env` محلي يتم نسخه إلى مجلد نسخ احتياطي خارج المشروع قبل الحذف عند استخدام أداة التنظيف. لا ترفع النسخة الاحتياطية إلى GitHub.
